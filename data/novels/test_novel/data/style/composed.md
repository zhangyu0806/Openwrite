# 最终风格文档：公司里的术师

> 合成时间：2026-03-04
> 提取风格源：无
> 来源：craft/ + 项目角色设定 + 项目运行态

## 1. 硬性约束（不可违反）

### 1.1 角色一致性

- **陈明**：28岁后端程序员，逻辑思维极强，习惯用编程概念理解术法（法阵=函数调用、灵力=内存资源、咒语=API接口），嘴上吐槽身体诚实——口头禅"这不科学"但每次都第一个冲上去研究，社恐但重情义，抗压能力惊人（996+修炼双线并行），格子衫+黑眼圈+冷掉的美式是视觉标签
- **赵磊**：26岁前端程序员，外向爱八卦，讲义气，是陈明公司里唯一能说真心话的人，日常负责把陈明从工位拖去吃饭
- **林月**：32岁技术组长，表面高冷实则护犊子，工作上要求严格但私下照顾组员，是"嘴硬心软"型上司

### 1.2 世界观规则

- 术师体系隐藏在现代社会，普通人不可知
- 术法表现为"异常现象"，可被科学解释掩盖（"服务器抖动""电磁干扰""静电反应"）
- 主角在公司里必须隐藏能力，觉醒印记用手表遮住
- 术师社会与现代社会平行运作，有自己的组织、规则和阶层
- 觉醒原因不明，陈明正处于术师体系入门阶段

### 1.3 禁忌清单

**禁用AI高频词**（来源：humanization.yaml）：

| 禁用 | 替代 |
|------|------|
| 然而 | 但是、不过、可、却 |
| 不禁 | 直接描述动作或感受 |
| 缓缓 | 慢慢、一点点 |
| 微微一笑 | 笑了笑、嘴角一歪 |
| 心中暗想 | 心想、琢磨着 |
| 眼中闪过一丝 | 直接描述眼神变化 |
| 深吸一口气 | 吸了口气 |
| 不由得 | 直接描述结果 |
| 难以置信 | 没想到、离谱 |
| 若有所思 | 想了想、琢磨着 |
| 刹那间/顿时 | 一下子、突然 |
| 油然而生/恍然大悟 | 心里生出…… / 想通了 |

**禁用网络流行语**：

- 绝绝子、yyds、芭比Q了、栓Q、家人们、蚌埠住了
- 绷不住了、笑死、无语子、真的会谢、i人/e人
- 任何当年度流行但生命周期短的热梗

**不可违反**：
- 角色性格不可突变（陈明不会突然变成话痨社牛）
- 世界观不可自相矛盾（普通人不可能直接看到术法本质）
- 不可出现不属于本书的异世界设定（监狱、血月、虚境等）

## 2. 风格约束（应当遵循）

### 2.1 叙述声音

**核心叙述目标，适配现代都市语境：**

- **第三人称有限视角**，紧贴陈明内心意识，几乎等同第一人称内心独白
- **叙述者-角色声音融合**（最重要的技巧）：叙述句与角色内心独白的边界故意模糊
  - 叙述句中直接使用角色口头禅和思维方式：
    > ✅「陈明盯着屏幕上的符文，脑子里第一反应是——这bug谁写的。」
    > ——"这bug谁写的"是陈明的思维，但出现在第三人称叙述中
  - 用角色会用的比喻做叙述描写：
    > ✅「灵力在经脉里跑了一圈，延迟大概两百毫秒，比公司内网还卡。」
  - 叙述者评论伪装成角色想法：
    > ✅「对于一个连需求文档都能debug出三个逻辑漏洞的人来说，术法体系的规则简直友好得不像话。」
- **核心原则**：读者应该分不清哪句是叙述者说的、哪句是陈明想的。如果能明确区分，说明融合度不够。
- 偶尔无缝切入第一人称内心独白，不加引号，直接用"我"：
  > ✅「所以我这是……在公司楼下画法阵？正常人不会干这事吧。算了，正常人也不会连续加班十六个小时。」
- **信息揭示策略**：
  - 先甜后打脸：先建立期望再颠覆（以为是升职加薪→其实是高危任务）
  - 行动先于解释：陈明先动手再理解后果
  - 旁观者误读：同事对陈明异常行为的错误解读制造喜剧（"你最近在练气功？"）
  - 通过行动/对话/阅读引入设定，禁止大段旁白式说明

### 2.2 语言风格

**语言目标：保留轻快口语化与精准类比，语境固定在现代 IT 公司：**

- **短段为主**：约60%为1-2句短段，保持轻快阅读节奏
- **口语化叙述**：轻松随意，带有聊天感，不用文绉绉的腔调
- **编程术语自然融入叙述**（替代原作的游戏/手游术语）：

| 编程概念 | 术法类比 | 示例 |
|----------|----------|------|
| 函数调用 | 法阵激活 | "这法阵的调用方式跟递归函数一样，弄不好就栈溢出" |
| 内存/资源管理 | 灵力消耗 | "灵力不够了，跟内存泄漏一个道理" |
| API接口 | 咒语 | "这咒语就是个API，入参不对直接报错" |
| debug | 修炼纠错 | "经脉堵了？让我debug一下" |
| 编译/部署 | 术法释放 | "编译通过了但运行时报错——术法炸了" |
| 版本管理 | 术法改良 | "这术法需要重构，初版写得太烂了" |
| 并发/多线程 | 多重施法 | "同时画两个法阵？我又不是多线程的" |

- **职场梗保留并强化**（从异世界职场隐喻→真实IT职场）：
  - 996、deadline、需求变更、产品经理、代码review
  - 年终奖、调薪、跳槽、面试
  - "这个需求做不了"→"这个术法学不会"的映射关系
- **标点符号习惯**（继承）：
  - 破折号"——"高频使用，每章5-15次
  - 省略号"……"用于犹豫、延伸
  - 感叹号+问号"！？"表达震惊+困惑

**修辞手法：**

1. **反差/解构（核心）**：用IT概念解读术法体系
   > ✅「所谓觉醒，大概就是系统给他推了一个版本更新，没有取消按钮那种。」
2. **刻意对比**：加班地狱与修炼地狱的直接比较
   > ✅「比起连续加班三天赶上线，修炼其实也没那么累——至少术法不会在凌晨三点给你提需求变更。」
3. **诗意-俗气切换**：美好描述后突然接地气
   > ✅「灵力流过全身的感觉像是清晨第一口冷空气——等等，他闻到了隔壁工位泡面的味道。」
4. **精准类比**：用程序员日常类比抽象概念
   > ✅「理解术法体系的感觉，跟看一份没有注释的祖传代码差不多。」

### 2.3 对话风格

**对话目标：结合 craft/dialogue_craft.md，适配现代都市语境：**

**格式规范：**
- 普通对话：`""` 双引号
- 内心独白：直接融入叙述，无引号，与叙述声音融合
- 手机消息/即时通讯：可用缩进+斜体或代码块风格呈现

**对话驱动模式：**

1. **乒乓球式信息对话**：每说1-2个信息点，对方必有反应
   ```
   "你手腕上那个纹身什么时候纹的？"赵磊探过头来。
   "……不是纹身。"
   "那是什么？过敏？"
   "嗯，过敏。"陈明把袖子往下拽了拽。
   赵磊狐疑地看了他一眼，"你对什么过敏能过敏成这个形状？"
   ```

2. **误会滚雪球对话**：同事基于正常认知解读陈明的异常行为
   ```
   "你最近是不是在学什么功夫？"
   "……为什么这么问。"
   "你昨天在楼梯间比划来比划去的。"
   "那是……在活动筋骨。"
   ```

3. **程序员式对话**：用技术语言聊日常
   ```
   "你这个方案的时间复杂度太高了。"林月扫了一眼他的文档。
   "这不能怪我，需求本身就是O(n²)的。"
   "优化。"
   "……是。"
   ```

4. **喜剧节奏**（继承）：
   - 期望→落空：看似深刻→实际很蠢/很俗
   - 冷场→补救失败：越描越黑
   - 伪深沉→暴露本性

**角色对话个性：**

| 角色 | 语言特征 | 典型表达 |
|------|----------|----------|
| 陈明 | 程序员口吻+技术类比+自嘲式吐槽+社恐回避 | "这不科学""我debug一下""能不能别@我" |
| 赵磊 | 八卦腔+夸张+直球+爱起哄 | "卧槽你不会吧""我跟你说啊""走走走吃饭去" |
| 林月 | 简洁精准+不废话+偶尔冷幽默+护短时语气软化 | "优化""下周上线""你们组谁也别加班了，都给我回去" |

**对话标签规则：**
- 两人快速对话省略标签
- 三人以上必须标签
- 优先用动作标签而非"说道"：
  > ✅ 赵磊把椅子滑过来，"你中午吃什么？"
  > ❌ 赵磊说道："你中午吃什么？"

### 2.4 幽默体系

**吐槽模式**（项目内风格要求，语境适配）：

| 类型 | 说明 | 示例 |
|------|------|------|
| **歪楼式吐槽**（核心） | 吐槽落点与触发点不在同一维度，故意跑题 | 触发：灵力暴走差点炸了工位 → 落点：「第一反应不是慌，而是——这个月绩效完了。」 |
| **程序员视角吐槽** | 用编程逻辑评价超自然现象 | 「这术法的容错率也太低了。写代码要是这么脆，早被review打回来了。」 |
| **社畜自嘲式** | 用打工人视角解构修炼 | 「修炼到凌晨两点，第二天还要九点打卡。996是福报，996+修炼才是真正的地狱。」 |
| **自我贬低式** | 主动降低期望值 | 「他已经放弃了'快速掌握'这个目标，改为'活着学完入门'。」 |
| **先甜后打脸** | 先建立期望再颠覆 | 「灵力运转完美，法阵成型——然后炸了。就像代码本地跑没问题，一上生产环境就崩。」 |
| **旁观者误读** | 同事对异常行为的合理化曲解 | 赵磊："你最近气色好了很多啊，在健身？" 陈明（昨晚修炼到三点）："……嗯。" |

**吐槽密度要求**：每500字至少1处吐槽，分布自然，不要集中堆积。

**幽默禁区**：
- 不用谐音梗/烂梗
- 不在严肃剧情高潮强行搞笑
- 吐槽之后要回归叙事主线，不能歪楼太远收不回来

## 3. 文本人化规则

**来源：craft/humanization.yaml，以下为关键规则摘要：**

### 3.1 自然不完美

| 允许的不完美 | 示例 |
|-------------|------|
| 片段句 | "行吧。" "算了。" "不想了。" |
| 重复用词 | "他就坐在那里，就坐在工位上。" |
| 口语化表达 | "那啥，你去不去？" |
| 不完整思考 | "如果……算了，不想了。" |
| 突然转折 | "他很认真。但其实并没有。" |

| 避免的完美 | 修正方式 |
|-----------|----------|
| 工整并列"第一……第二……" | 打散为自然叙述 |
| 每段都有过渡词 | 省略部分过渡 |
| 段落长度均衡 | 长短参差 |
| 每个想法都有完整解释 | 留白让读者脑补 |

### 3.2 句式变化

| 句子类型 | 长度 | 目标比例 |
|---------|------|---------|
| 短句 | < 10字 | 30-40% |
| 中句 | 10-25字 | 40-50% |
| 长句 | > 25字 | 15-20% |

- 避免连续3+句以"他/她"开头
- 避免连续3+句以"然后/于是/接着"开头
- 开头模式要变化，不可重复

### 3.3 情感表达

- **展示而非告知**：
  > ❌ 他感到紧张。
  > ✅ 他的手指在键盘上悬了两秒，没落下去。
- **克制煽情**：煽情段落后必须接冷静转折或日常吐槽
- 用一个精准词代替多个模糊形容词

### 3.4 对话自然化

- 每个角色对话有区分度（见2.3角色对话个性表）
- 两人快速对话省略标签
- 乒乓球规则：信息提供者每说1-2个信息点，接收者必须有反应
- 禁止连续灌输超过3段无反应

- [ ] 无AI高频词（然而、不禁、缓缓、微微一笑、心中暗想等）

- [ ] 短句（<10字）占比≥30%

- [ ] 角色声音一致（陈明=程序员吐槽、赵磊=八卦直球、林月=简洁精准）

- [ ] 煽情段落有冷静转折

- [ ] 章末有悬念钩子
