# 通用场景结构技法 (Universal Scene Craft)

> 这里记录的是结构模式，不是某部作品的复刻模板。

## 一、设定说明场景

### 核心结构

```text
信息点A → 角色反应 → 信息点B → 追问/误解 → 信息点C → 新问题
```

### 规则

| 规则 | 说明 |
|------|------|
| 3 段上限 | 连续解释不超过 3 段就要有人反应 |
| 讲解即推进 | 讲设定时也要推进关系、冲突或判断 |
| 反应多样化 | 不能总是“原来如此” |
| 绑定人物性格 | 同一设定由不同角色讲，质感应不同 |

### 常见变体

| 变体 | 适用场景 |
|------|----------|
| 导师问答式 | 知识传递、入门教学 |
| 日常嵌入式 | 工作、吃饭、路途中自然引出 |
| 训话/动员式 | 组织会议、长辈训诫、战前部署 |
| 类比映射式 | 新规则通过旧经验类比理解 |

## 二、战斗/冲突场景

### 核心结构

```text
对峙铺垫 → 试探 → 危机 → 转机 → 爆发 → 清算
```

### 规则

| 规则 | 说明 |
|------|------|
| 战中必须有判断 | 不能只剩动作播报 |
| 旁观视角分层 | 外行、内行、利益相关者看到的重点不同 |
| 代价明确 | 体力、资源、关系、时间，总要损失点什么 |
| 战后要结算 | 收获、风险、后患都要落地 |

### 常见变体

| 变体 | 特征 |
|------|------|
| 暴力美学式 | 环境与感官渲染强，爆发后静默 |
| 策略式 | 以信息差、布置和误导取胜 |
| 成长反馈式 | 冲突后伴随清晰成长或认知变化 |
| 考验通关式 | 标准答案 vs 非标准解法 |

## 三、日常场景

### 核心结构

```text
熟悉空间 → 人物聚集 → 轻松互动 → 温馨细节 → 平静收尾或突发打断
```

### 隐藏功能

- 给高压剧情降速
- 埋伏笔
- 推关系
- 展示世界的生活层

### 规则

- 固定空间能形成情感锚点
- 食物、器物、工作细节比空泛抒情更有效
- 日常不是“无事发生”，而是低压推进

## 四、情感高潮场景

### 核心结构

```text
细节铺垫 → 情感触发 → 爆发 → 收束/打断 → 余韵
```

### 规则

| 规则 | 说明 |
|------|------|
| 煽情不过量 | 连续高浓度抒情不宜过长 |
| 细节胜过宣言 | 用动作、物件、停顿传达情绪 |
| 余韵要留白 | 不必把所有感受解释完 |
| 收尾宜朴素 | 日常化收尾通常更有力 |

## 五、说服/谈判场景

### 核心结构

```text
观察对方 → 建立共识 → 逐层推进 → 击中核心 → 对方动摇 → 达成或破局
```

### 规则

- 先共情，再上逻辑
- 给选择，但让最优解清晰
- 让话术背后有真正利益
- 威压不一定靠声量，也可以来自局势

## 六、反转/揭示场景

### 核心结构

```text
预期建立 → 线索累积 → 反转揭晓 → 重新解释前文
```

### 规则

- 反转前必须给足线索
- 反转后要能回看成立
- 不要只追求“意外”，还要追求“合理”

## 七、场景组合节奏

推荐把不同强度场景交替安排：

- 设定说明后接人物反应或日常
- 高强冲突后给消化空间
- 情感高潮后用短收尾落地
- 长谈判前先给清晰 stakes

## 场景自检

- [ ] 这一场真正改变了什么
- [ ] 是否有明确的前进力
- [ ] 是否有多余解释可以砍
- [ ] 是否能分清主视角人物的目标
